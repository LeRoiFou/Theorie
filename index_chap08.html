Chapitre VIII : Sauvegardes & Monitoring

💾 1. Les sauvegardes (Backups)

    Pourquoi ?

  • Protéger les données contre :
    • Panne matérielle ⚡
    • Piratage 🦹
    • Erreur humaine 🙈
  • Sans sauvegarde → risque de perte totale des données

    • Types de sauvegardes :

  • Complète → copie intégrale du système
  • Incrémentale → copie uniquement des changements depuis la dernière sauvegarde
  • Différentielle → copie des changements depuis la dernière sauvegarde complète

    • Bonnes pratiques (3-2-1 Rule) :

  • 3 copies de tes données
  • 2 supports différents (ex. disque + cloud)
  • 1 copie hors site (ex. cloud ou autre data center)

📡 2. Le monitoring (surveillance)

    Pourquoi ?

  • S’assurer que le serveur fonctionne bien en continu
  • Détecter rapidement :
    • une surcharge CPU/RAM
    • un disque presque plein
    • un service qui tombe (site inaccessible)

    Outils courants :

  • Nagios, Zabbix, Prometheus, Grafana → monitoring avancé
  • UptimeRobot → simple vérification si un site est en ligne
    • Indicateurs surveillés :
  • CPU, RAM, stockage
  • Disponibilité (uptime)
  • Trafic réseau
  • Logs pour repérer les erreurs

📖 3. Analogie simple
  • Sauvegarde = assurance habitation 🏠 : si tu perds, tu reconstruis
  • Monitoring = tableau de bord 🚗 : tu surveilles l’état de ton véhicule en temps réel
